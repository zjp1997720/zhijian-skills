# skill-open-sourcer characterization

- **Trigger contract:** Package and publish a local Skill as a safe, documented, installable GitHub release.
- **Safety contract:** Secrets, private paths, client data, proprietary material, and ambiguous ownership block publishing.
- **Workflow invariant:** Resolve source, check environment, scan safety, build package, audit README, verify install, then publish.
- **Output invariant:** A clean public package plus release documentation and launch copy.
- **Resource graph:** release/safety references, environment and safety scanners, README auditor, and tests.
