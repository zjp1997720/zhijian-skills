# 结构化提取规范

从原始素材提取企业画像和内容资产的操作指南。

## 提取原则

1. **只提取有证据的内容**：每条信息必须能在原始素材中找到出处
2. **区分证据和推断**：有明确原文支撑的标"高"，单一来源的标"中"，推断的标"低"或"待核实"
3. **不编造**：素材里没有的不能编。没有客户名就用"应用场景"描述
4. **来源标注**：每条关键事实标注来源文件路径
5. **矛盾处理**：多个来源矛盾时，本地资料优先于联网资料；都标注出来让用户判断

## 01-企业画像 提取规范

### 01-基本信息.md

提取字段：
- 企业全称、法定代表人、注册资本、成立时间
- 企业类型、企业性质
- 注册地址、办公地址（不一致时都标注）
- 联系方式：电话、传真、邮箱、邮编、官网
- 经营范围（工商登记版）
- 资质荣誉（只提取有证书证据的）
- 员工/参保人数（如有）

来源优先级：本地工商文件 > 官网页脚 > 第三方平台 > 交接包

### 02-产品与服务.md

提取结构：
```
## 产品线分类

### {产品线名称}
- 核心型号
- 关键技术参数（量程、精度、容量等）
- 执行标准（GB/ISO等）
- 应用场景
- 核心卖点
- 来源：{文件路径}
```

每个产品线一个章节。如果有产品手册，以产品手册为准；没有则从官网产品页提取。

### 03-客户与市场.md

提取内容：
- 目标客户类型（管材厂/检测机构/科研院所等）
- 行业分布
- 地域市场（国内区域、是否有出口）
- 竞争格局（如有行业报告）
- 具体客户名（如果有且可公开）

严格区分"有证据的客户"和"推断的客户类型"。

### 04-品牌调性.md

提取内容：
- 品牌定位（企业怎么自我定位）
- 品牌声音（专业/务实/技术导向/其他）
- 表达习惯：常用词汇、句式、自称方式、收尾方式
- 产品描述风格
- 禁用表达

来源：官网文案、公众号文章、宣传册、第三方平台介绍。

### 05-发展历程.md

提取内容：
- 成立时间
- 有时间证据的关键节点（产品发布、业务扩展、重大事件）
- 发展阶段划分

注意：没有时间证据的节点不能编造。2018-2020年如果没有素材，标注"可能是抓取覆盖不足，不可断言企业停滞"。

## 03-内容资产 提取规范

### 产品描述库.md

每个产品一个结构化条目：

```
## {产品名称}

| 字段 | 内容 |
|---|---|
| 型号 | |
| 产品线 | |
| 定位 | 一句话说明做什么 |
| 关键参数 | |
| 执行标准 | |
| 应用场景 | |
| 核心卖点 | |
| 来源 | {文件路径} |
```

### 客户案例库.md

如果没有具体客户名（大概率没有）：
- 整理应用场景分类
- 每个场景的典型工作流程
- 配套设备
- 明确标注"无具体客户名称，基于应用场景推断"

### 技术知识库.md

从技术文档和行业资料提取：
- 行业背景
- 核心技术原理
- 检测/服务标准索引
- 选型知识（如有"如何选购"类内容）

### 品牌素材.md

- Slogan（如有；没有则提取候选方向）
- 企业简介：标准版（300字）+ 短版（100字）
- 产品总述
- 资质文案（只写有证据的）
- 营销话术
- 联系方式标准文案

## claims-ledger.md 写法

每条断言一行：

```markdown
| claim | type | source | confidence | notes |
|---|---|---|---|---|
| 企业成立于2013年 | 工商信息 | 机电之家企业页 | 中 | 单一来源，建议工商年报复核 |
```

置信度标准：
- **高**：多个独立来源一致，或官方一手来源
- **中**：单一来源，未交叉验证
- **低**：推断或来源可信度存疑
- **待核实**：有但无法确认

## gap-report.md 写法

分两类记录：

**已解决缺口**：建库过程中已填充的，标注填充位置。

**仍存在缺口**：标注影响和解决路径。按高/中/低优先级排序。
