# 标准目录结构规范

所有企业分身使用统一目录结构。builder 按此创建，writer 按此读取。

## 完整结构

```

  README.md
```

## 层级职责

| 层级 | 目录 | 职责 |
|---|---|---|
| 根目录 | 3个md + 目录入口 | 身份入口和速查 |
| 01-企业画像 | 5个md | 理解层：企业身份的5个维度 |
| 02-原始素材 | 原始文件归档 | 证据层：只读归档，保证可追溯 |
| 03-内容资产 | 4个md | 可用层：结构化提取的可复用内容 |
| 04-文风样本 | anchors+cues+normalized | 风格层：文风锚点和特征 |
| 05-调研记录 | 4个md | 元信息层：质量保障 |
| 06-产出 | 空目录 | writer的输出位置 |

## 文件 frontmatter 规范

所有 md 文件使用统一 frontmatter：

```yaml
---
status: draft | complete
source: 公开信息整理 | 本地资料整理 | 混合
last_updated: YYYY-MM-DD
---
```

- `status: draft`：骨架或部分填充
- `status: complete`：已验证填充完整

## 02-原始素材/本地资料/ 子目录

| 子目录 | 接收文件 |
|---|---|
| 企业介绍/ | 公司简介、发展历程、团队介绍、企业文化、资质荣誉 |
| 产品资料/ | 产品手册、技术参数表、产品目录、报价单、服务说明 |
| 客户案例/ | 客户名单、合作案例、项目成果、客户评价 |
| 对外发声/ | 公众号文章、官网新闻、宣传册、白皮书、讲话稿 |
| 销售服务/ | 销售话术、客服话术、FAQ、投标文件模板 |
| 行业知识/ | 行业报告、技术标准、竞品分析、市场调研 |
| 品牌视觉/ | Logo、VI手册、标准色规范 |
| 未分类/ | 无法确定类型的文件 |

## 06-产出/ 目录

初始为空。enterprise-clone-writer 的所有写作产出放这里。

文件命名：`{YYYY-MM-DD}_{主题关键词}_{原型}.md`
