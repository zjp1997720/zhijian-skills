# 文风分析操作指南

## 选文标准

从对外发声类素材中选 3-5 篇代表性文章。

### 优先级

本地公众号文章 > 官网新闻 > 第三方平台介绍 > 产品详情页

### 选择标准

1. **覆盖不同类型**：至少包含1篇企业动态 + 1篇技术文章 + 1篇产品介绍
2. **优先近期**：近2年的文章优先（文风可能有演变）
3. **内容丰富**：选有实质内容的，跳过纯通知、纯招聘、一句话新闻
4. **字数适中**：500字以上的文章才有文风分析价值

### 如果素材不足

- 只有1-2篇：标注"文风样本不足，分析基于有限素材"
- 0篇：标注"无对外发声素材，文风分析无法进行，建议企业补充历史公众号文章"
- 用产品详情页的描述文字作为最低保底素材

## voice-anchors.md 写法

收录选定的3-5篇文章，每篇记录：

```markdown
## 锚点N：{文章标题}

- 来源：{URL或文件路径}
- 发布时间：{日期}
- 类型：企业动态 | 技术文章 | 产品介绍 | 选型指导 | 其他
- 文风标注：这篇文章体现了{什么文风特征}

{文章全文或关键段落}
```

## voice-cues.md 提取维度

从锚点文章蒸馏以下维度，**每条必须附原文证据**：

### 1. 语气与人设

- 企业的"人格"是什么？（技术专家？老厂长？销售工程师？）
- 对读者的态度（平等/权威/服务/教育）
- 情绪基调

附原文示例。

### 2. 自称方式

- 企业怎么称呼自己？（全称？简称？"我公司"？"我们"？）
- 标题里怎么自称？
- 附件出处怎么署名？

### 3. 句式与节奏

- 常用句式模板（从原文提取真实例子）
- 句子长度偏好（短句为主？长句为主？混合？）
- 段落结构特征

### 4. 用词特征

- 高频技术词汇
- 常用表达/口头禅
- 行业术语处理方式（直接用？加解释？）

### 5. 结构偏好

- 文章开头怎么写（定义？痛点？新闻？观点？）
- 中段怎么展开（分点？对比？案例？叙述？）
- 结尾怎么收（总结？行动号召？联系方式？）

### 6. 产品描述风格

- 怎么描述产品参数（列表？嵌入叙述？）
- 怎么表达产品优势
- 怎么处理标准引用

### 7. 禁用/避免

- 基于品牌调性，哪些表达方式不适合该企业
- 官网素材中有但新内容不应继承的瑕疵（SEO残留、错别字等）

## form-normalized-anchors 写法

选1-2篇最有代表性的文章做归一化处理：

- 保留原文核心内容和文风
- 清理HTML残留、格式混乱
- 统一标题层级
- 保留frontmatter记录原文URL

文件命名：`anchor-01-{主题关键词}-规整版.md`

## 文风置信度标注

在 voice-cues.md 末尾标注：

```markdown
## 文风置信度

- 样本数：{N篇}
- 置信度：高（5篇以上，覆盖多类型）/ 中（3-4篇）/ 低（1-2篇）
- 时间跨度：{最早} 至 {最晚}
- 注意事项：{如有文风演变、样本偏窄等问题}
```

## 文风演变的处理

如果发现早期文章和近期文章文风明显不同（如东海试验机2016年偏宣传、2025年偏学术化），在 voice-cues.md 里分段记录：

```markdown
### 早期文风（{年份范围}）
...

### 近期文风（{年份范围}）
...

### 建议
新内容以{近期/早期}文风为基准。
```
